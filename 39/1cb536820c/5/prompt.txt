# Batch Delete Subscriptions

## Context

The app supports batch-creating subscriptions but not batch-deleting them. Adding a `DELETE /api/v1/subscriptions/batch` endpoint allows the frontend to delete multiple subscriptions atomically.

## Files to Modify

### Backend
1. `internal/api/subscriptions.go` — Add `BatchDeleteInput` struct, `BatchDelete` handler, extend `SubscriptionsStore` interface
2. `internal/api/pgstore.go` — Add `DeleteSubscriptionBatch` using `DELETE ... WHERE id = ANY($1)`
3. `internal/api/server.go` — Register `DELETE /api/v1/subscriptions/batch` route
4. `internal/api/subscriptions_test.go` — Add mock method + tests for batch delete

### Frontend
5. `web/lib/api/types.ts` — Add `BatchDeleteSubscriptionInput` type
6. `web/lib/api/client.ts` — Add `batchDelete` method to subscriptions client

### Docs
7. `API.md` — Add batch delete endpoint + request example

## Implementation

### Step 1: Backend interface + handler (`subscriptions.go`)

Add to `SubscriptionsStore`:
```go
DeleteSubscriptionBatch(ctx context.Context, ids []string) error
```

Add input struct:
```go
type BatchDeleteSubscriptionInput struct {
    IDs []string `json:"ids"`
}
```

Add `BatchDelete` handler: decode JSON, validate `ids` non-empty, call store, return `204 No Content`.

### Step 2: Store implementation (`pgstore.go`)

```go
func (s *PgStore) DeleteSubscriptionBatch(ctx context.Context, ids []string) error {
    tag, err := s.pool.Exec(ctx, `DELETE FROM subscriptions WHERE id = ANY($1)`, ids)
    // check tag.RowsAffected() > 0
}
```

Single query with `ANY($1)` — no transaction needed for a single DELETE statement.

### Step 3: Route registration (`server.go`)

Add `DELETE /api/v1/subscriptions/batch` — must be registered before `DELETE /api/v1/subscriptions/{id}` to avoid route conflict.

### Step 4: Tests (`subscriptions_test.go`)

- Add `batchDeleteErr` and `batchDeletedIDs` to mock store
- Implement `DeleteSubscriptionBatch` on mock
- Tests: success case, empty IDs (422), invalid JSON (400)

### Step 5: Frontend types (`types.ts`)

```ts
export interface BatchDeleteSubscriptionInput {
  ids: string[];
}
```

### Step 6: Frontend client (`client.ts`)

```ts
batchDelete: (input: BatchDeleteSubscriptionInput) =>
  request<ApiResponse<null>>("/subscriptions/batch", {
    method: "DELETE",
    body: JSON.stringify(input),
  }),
```

### Step 7: API.md

Add row to subscriptions table and request body example.
