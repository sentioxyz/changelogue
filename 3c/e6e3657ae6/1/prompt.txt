# Plan: Merge "Semantic Releases" into "Releases"

## Context

The dashboard currently has two separate concepts — "Releases" (source-level facts) and "Semantic Releases" (AI-generated project-level reports). This creates navigation clutter (two sidebar items, two separate pages) and cognitive overhead. Since every release can optionally have a semantic report, we merge them into a single "Releases" concept where each release row shows inline report status.

## Goal

- Remove the standalone "Semantic Releases" page and sidebar item
- Enrich each release row with its semantic report status (none / in-progress / completed)
- Show trigger/spinner/link inline on the Releases page and Projects page

Add 3 fields to `Release` struct:

```go
SemanticReleaseStatus string `json:"semantic_release_status,omitempty"`
SemanticReleaseUrgency string `json:"semantic_release_urgency,omitempty"`
```

### `internal/api/pgstore.go`
Modify all 4 release query methods to LEFT JOIN `semantic_releases` on `(project_id, version)`:

**`ListAllReleases`** (2 queries at lines 230-253): Add to each SELECT:

```sql
LEFT JOIN LATERAL (
```

Add columns: `COALESCE(sr_info.id::text,''), COALESCE(sr_info.status,''), COALESCE(sr_info.urgency,'')`

**`ListReleasesByProject`** (2 queries at lines 363-404): Same pattern. Note: these use `JOIN` not `LEFT JOIN` for sources/projects, so `p.id` is always available.
