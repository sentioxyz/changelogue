# Switch GitHub Source from Atom Feed to REST API

## Context

The GitHub source currently uses the Atom feed (`/releases.atom`) which doesn't include the `prerelease` boolean. To support "Exclude Pre-Releases" for GitHub sources, we need to switch to the GitHub REST API (`/repos/{owner}/{repo}/releases`) which provides `prerelease`, `draft`, `body` (markdown changelog), and richer metadata.

The `prerelease` field will be stored in the release's `raw_data` JSONB as `"prerelease": "true"/"false"`. Users can then use the existing `version_filter_exclude` with a separate mechanism — or we add a dedicated boolean toggle on the source. Since the user's description showed a dedicated "Exclude Pre-Releases" toggle, we'll add an `exclude_prereleases` boolean to the `Source` model.

## Plan

### 1. Replace `github_atom.go` with `github.go` (REST API)

**Files:** Create `internal/ingestion/github.go`, delete `internal/ingestion/github_atom.go`

- New `GitHubSource` struct (replaces `GitHubAtomSource`) hitting `https://api.github.com/repos/{owner}/{repo}/releases?per_page=30`
- Parse JSON response array: extract `tag_name` (version), `body` (changelog), `html_url`, `published_at`, `prerelease`, `draft`
- Skip `draft` releases entirely

- Constructor: `NewGitHubSource(client, repository, sourceID)`
- Support optional `GITHUB_TOKEN` env var for auth (set `Authorization: Bearer` header if present) — avoids rate limits
