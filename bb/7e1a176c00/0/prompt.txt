# Replace "Needs Attention" Card with Release Trend Chart

## Context

The "Needs Attention" stat card counts semantic releases with critical/high urgency, but has no mechanism to clear/acknowledge items — the number only grows over time. We're replacing it with a release trend bar chart that shows the velocity of releases and AI reports over time, with switchable daily/weekly/monthly granularity.

## Approach

- New backend endpoint `GET /api/v1/stats/trend?granularity={daily|weekly|monthly}` returns time-bucketed counts using PostgreSQL `generate_series` + `date_trunc`
- Install `recharts` for the bar chart (most popular React charting lib, good Tailwind/inline-style compat)
- Remove the 3rd stat card, keep 2 stat cards stacked vertically beside the chart
- Layout: `[2 stat cards | trend chart]` side-by-side on desktop, stacked on mobile

## Backend endpoint

**URL:** `GET /api/v1/stats/trend?granularity={daily|weekly|monthly}`

**Response:**
```json
{
  "data": {
    "granularity": "daily",
    "buckets": [
      { "period": "2026-01-29", "releases": 3, "semantic_releases": 1 },
      { "period": "2026-01-30", "releases": 0, "semantic_releases": 0 }
    ]
  }
}
```

**Ranges:** daily=30 days, weekly=12 weeks, monthly=12 months

**SQL approach:** `generate_series` for zero-filled buckets, LEFT JOIN against `releases` (using `COALESCE(released_at, created_at)`) and `semantic_releases` (using `created_at`). Granularity validated to 3 string literals before query — no injection risk.

     func (s *PgStore) GetTrend(ctx context.Context, granularity string, start, end time.Time) ([]TrendBucket, error) {
         var trunc string
         switch granularity {
         case "daily": trunc = "day"
         case "weekly": trunc = "week"
         case "monthly": trunc = "month"
         default: return nil, fmt.Errorf("invalid granularity: %s", granularity)
         }
         query := fmt.Sprintf(`
             WITH buckets AS (
                 SELECT date_trunc('%s', gs)::date AS period
                 FROM generate_series($1::timestamptz, $2::timestamptz, '1 %s'::interval) gs
             )
             SELECT b.period, COALESCE(r.cnt, 0), COALESCE(sr.cnt, 0)
             FROM buckets b
             LEFT JOIN (...releases subquery...) r ON r.period = b.period
             LEFT JOIN (...semantic_releases subquery...) sr ON sr.period = b.period
             ORDER BY b.period
         `, trunc, trunc, trunc, trunc)

     }
