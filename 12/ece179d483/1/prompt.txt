# Fix Dark Mode & i18n Across All Pages

## Context

After implementing the settings dialog with theme toggle (light/dark/system) and language selector (EN/ZH), users reported:
1. **Dark mode broken**: Cards, buttons, tables still show white/light backgrounds because components use hardcoded hex colors instead of CSS variables
2. **i18n incomplete**: Only sidebar nav labels are translated — all page content (titles, tabs, buttons, table headers, messages) remains hardcoded English

This plan systematically replaces all hardcoded colors with CSS variables and adds translation keys across all ~30 affected files.

## Strategy

### Colors: Hardcoded hex → CSS variable mapping

| Hardcoded hex | Replace with | Notes |
|---|---|---|
| `#ffffff`, `bg-white` | `var(--surface)` / `bg-surface` | Card/panel backgrounds |
| `#fafaf9` | `var(--background)` / `bg-background` | Page/section backgrounds |
| `#f3f3f1` | `var(--mono-bg)` / `bg-mono-bg` | Code blocks, badges, muted surfaces |
| `#111113` | `var(--foreground)` / `text-foreground` | Primary text |
| `#374151` | `var(--secondary-foreground)` / `text-secondary-foreground` | Secondary text (darker gray) |
| `#6b7280` | `var(--text-secondary)` / `text-text-secondary` | Secondary text |
| `#9ca3af` | `var(--text-muted)` / `text-text-muted` | Muted/placeholder text |
| `#b0b0a8`, `#c4c4c0` | `var(--text-muted)` | Light icons/decorative |
| `#e8e8e5` | `var(--border)` / `border-border` | Borders |
| `#e8601a` | `var(--beacon-accent)` | Accent (same in both themes) |
| `#fff7ed` | `var(--beacon-accent)` with `/10` opacity | Light accent surfaces |
| `#f8f8f6` | `var(--background)` | Auth loading background |
| `rgba(107,114,128,0.08)` | `var(--muted)` | Subtle badges |

For inline styles: use `var(--variable-name)`
For Tailwind classes: use `bg-surface`, `text-foreground`, `border-border`, etc.

### i18n: Add keys to en.json/zh.json, use `t()` in components

Group translation keys by page/component. Components that need `t()` must be client components (add `"use client"` if missing, or pass translated strings as props from client parent).
