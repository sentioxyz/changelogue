# Plan: Add OpenAI Model Support

## Context

Agent runs currently only work with Gemini (hardcoded `gemini-2.5-flash`). The user wants to use OpenAI models (e.g. `gpt-5.2`) as an alternative provider. We'll write a lightweight in-repo OpenAI adapter implementing the ADK `model.LLM` interface, and add a factory to switch between providers via env vars.

Reference: [jiatianzhao/adk-go-openai](https://github.com/jiatianzhao/adk-go-openai) for conversion patterns.

## Steps

### 1. Create `internal/agent/openai/openai.go` — OpenAI adapter

New file implementing `model.LLM` interface (2 methods: `Name()`, `GenerateContent()`).

Key responsibilities:
- Convert `req.Config.Tools` (`[]*genai.Tool` containing `FunctionDeclaration`s) → OpenAI `tools` array format
- Convert `req.Contents` (`[]*genai.Content`) → OpenAI messages:
  - Role `"user"` → `"user"`, role `"model"` → `"assistant"`
  - `Part.Text` → message content string
  - `Part.FunctionCall` → assistant message with `tool_calls` array
  - `Part.FunctionResponse` → `"tool"` role message with `tool_call_id`
- Convert `req.Config.SystemInstruction` → `"system"` role message
- Map config fields: `Temperature`, `TopP`, `MaxOutputTokens`, `StopSequences`
- Make HTTP POST to `{baseURL}/chat/completions`
- Convert response back to `*model.LLMResponse` with `genai.Content`:
  - Text → `genai.Part{Text: ...}`
  - `tool_calls` → `genai.Part{FunctionCall: &genai.FunctionCall{ID, Name, Args}}`
  - `finish_reason` → `genai.FinishReason`
- Non-streaming only (stream=false returns single-yield iterator, stream=true same behavior)
- Include `Close()` method (no-op, no background goroutines needed since we don't track sessions)

### 2. Create `internal/agent/model.go` — Model factory

New file with factory function:

```go
type LLMConfig struct {
    Provider string // "gemini" or "openai"
    Model    string // e.g. "gemini-2.5-flash", "gpt-5.2"
    // Gemini
    GoogleAPIKey string
    // OpenAI
    OpenAIAPIKey  string
    OpenAIBaseURL string
}

func NewLLMModel(ctx context.Context, cfg LLMConfig) (model.LLM, error)
```

Switches on `cfg.Provider`, calls either `gemini.NewModel()` or `openai.NewModel()`.

### 3. Modify `internal/agent/orchestrator.go`

- Change `Orchestrator` struct: replace `apiKey string` with `llmConfig LLMConfig`
- Change `NewOrchestrator(store, cfg LLMConfig)` signature — accept config instead of reading env vars internally
- Change `executeAgent()`: call `NewLLMModel(ctx, o.llmConfig)` instead of hardcoded `gemini.NewModel()`
- Remove `defaultModel` constant (model name now comes from config)

### 4. Modify `cmd/server/main.go`

- Read new env vars: `LLM_PROVIDER`, `LLM_MODEL`, `OPENAI_API_KEY`, `OPENAI_BASE_URL`
- Build `LLMConfig` and pass to `NewOrchestrator(pgStore, llmConfig)`
- Default: `provider=gemini`, `model=gemini-2.5-flash`

### 5. Update `CLAUDE.md`

Add new env vars to the environment variables table.
