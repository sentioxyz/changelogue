# Multi-Select Delete UI for Subscriptions Page

## Context

The batch delete backend (`DELETE /api/v1/subscriptions/batch`) and frontend client (`subsApi.batchDelete`) are implemented, but the subscriptions page only supports one-at-a-time delete. Need checkbox multi-select with a bulk delete action bar.

## File to Modify

- `web/app/subscriptions/page.tsx` — the only file that needs changes

## Implementation

### 1. Add selection state

```ts
const [selectedIds, setSelectedIds] = useState<Set<string>>(new Set());
```

Helpers:
- `toggleSelect(id)` — add/remove single ID
- `toggleSelectAll()` — select all / deselect all
- `isAllSelected` / `isSomeSelected` — derived booleans

### 2. Add checkbox column to table

- **Header row**: Add a checkbox cell before "Type". Uses `checked={isAllSelected}` or `"indeterminate"` when partially selected.
- **Each data row**: Add a checkbox cell bound to `selectedIds.has(sub.id)`.

Reuse existing `Checkbox` from `@/components/ui/checkbox` (Radix, supports `"indeterminate"` state).

### 3. Add selection action bar

When `selectedIds.size > 0`, render a bar between the page header and the table:

```
  N selected                           [Delete Selected]
```

- "Delete Selected" button with destructive styling opens a `ConfirmDialog` (already used for single delete)
- On confirm: `subsApi.batchDelete({ ids: [...selectedIds] })` → mutate → clear selection

### 4. Clear selection after mutations

After create, edit, single delete, and batch delete — reset `selectedIds` to empty set.

## Existing code to reuse

- `Checkbox` — `web/components/ui/checkbox.tsx`
- `ConfirmDialog` — `web/components/ui/confirm-dialog.tsx`
- `subsApi.batchDelete` — `web/lib/api/client.ts`
- `BatchDeleteSubscriptionInput` — `web/lib/api/types.ts`
